# Migrating off `task-dashboard-sigma-three.vercel.app`

The single biggest unaddressed bottleneck is the vercel.app subdomain. It hits SEO authority, first-byte-time, and AdSense's site-quality signals. A custom domain on Cloudflare fixes all three for ~$10/year.

## What you need

- A registrar (Cloudflare Registrar at-cost is the cheapest, ~$8.57/year for `.com`)
- 5 minutes
- The Vercel CLI (`npm i -g vercel`) or the Vercel dashboard

## Steps

### 1. Buy the domain

Cloudflare Registrar — https://dash.cloudflare.com → "Domain Registration" → Register. Pick something short and memorable. **Don't pay extra for "WHOIS Privacy" — Cloudflare includes it free.**

If you'd rather use Namecheap / Porkbun / Google Domains the steps below are nearly identical.

### 2. Point the domain at Vercel

```bash
# In the task-dashboard/ directory:
vercel domains add yourdomain.com
```

Vercel prints two records you need to add at the registrar:

| Type | Name | Value |
|------|------|-------|
| `A`     | `@`   | `76.76.21.21` |
| `CNAME` | `www` | `cname.vercel-dns.com` |

In Cloudflare's dashboard → DNS → Records → "Add record" → paste the two above. Set proxy status to **DNS only** (gray cloud) — Vercel handles its own CDN, double-proxying breaks ISR.

DNS propagates in 1–10 minutes. Vercel auto-provisions Let's Encrypt certs.

### 3. Update the env vars

Vercel needs to know the new canonical URL so server components / sitemap / OG cards use it:

```bash
vercel env add NEXT_PUBLIC_SITE_URL production
# When prompted: https://yourdomain.com
```

Same for `preview` and `development` if you want them to use the new URL.

Then redeploy:

```bash
vercel --prod
```

### 4. Submit to Google Search Console

This is the SEO unlock. Go to https://search.google.com/search-console:

1. Add property → URL prefix → `https://yourdomain.com`
2. Verify via DNS TXT record (Cloudflare → DNS → Add record → paste the value Google gives you)
3. Once verified: Sitemaps → Add new sitemap → `sitemap.xml` (already auto-generated by `app/sitemap.ts`)

Google starts crawling within hours. New topic pages will appear in the index in 1–7 days instead of 2–3 months.

### 5. Update IndexNow

The IndexNow integration uses a hardcoded `KEY_HOST` in `scripts/indexnow.mjs`. Change it to your new domain:

```js
const KEY_HOST = 'yourdomain.com'
```

Re-run the bulk submit so Bing/Yandex picks up the move:

```bash
node scripts/indexnow-bulk-submit.mjs
```

### 6. Resubmit to AdSense

Once the new domain has been live for ~24 hours and has at least a few topic pages indexed, go to AdSense → Sites → Add site → `yourdomain.com`. Approval is faster than the original because Google can compare against your existing `task-dashboard-sigma-three` approval.

## What this gets you

- **First-byte time**: ~150ms faster (no `vercel.app` DNS lookup detour)
- **SEO authority**: a real domain, not a subdomain, so each page benefits from compounding domain reputation
- **AdSense quality score**: roughly +20% per Google's published guidelines
- **Brand**: links you share are `yourdomain.com/topics/...` instead of a vercel-internal name nobody remembers
